PLAN: Add gamiphy.ai Dataset Upload to lerobot
================================================

Goal
----
Add a `push_to_gamiphy` option to lerobot's recording pipeline that uploads
datasets to gamiphy.ai after recording completes. Users authenticate via an
API key stored in the GAMIPHY_UPLOAD_KEY environment variable.

Upload mechanism: pre-signed URLs
  1. Call gamiphy.ai API to create a dataset record → get dataset_id + map of
     {relative_file_path: signed_upload_url}
  2. PUT each local file to its signed URL (streaming, no auth header needed)
  3. Call gamiphy.ai API to confirm upload → dataset appears in dashboard

Auth: Authorization: Bearer <GAMIPHY_UPLOAD_KEY>

Post-upload: metadata registered in Supabase DB, dataset visible in gamiphy.ai dashboard.


Files to Create
---------------

1. src/lerobot/utils/gamiphy.py
   Self-contained HTTP client module. No lerobot imports; depends only on stdlib
   and `requests`. Contents:

   a) GamiphyUploadError(RuntimeError)
      Custom exception raised at every failure point with human-readable messages.

   b) get_gamiphy_api_key() -> str
      Reads GAMIPHY_UPLOAD_KEY env var. Raises GamiphyUploadError immediately
      if missing or empty (fast-fail before any I/O).

   c) collect_dataset_files(root: Path) -> list[Path]
      Globs the dataset root directory for files to upload:
        data/**/*.parquet
        meta/*.json
        meta/*.parquet
        meta/episodes/**/*.parquet
        videos/**/*.mp4
      Excludes: images/ (raw PNG intermediates, never pushed upstream)
      Returns sorted list of absolute Paths.

   d) GamiphyClient class
      __init__(api_key: str, base_url: str = "https://api.gamiphy.ai")
        - Creates requests.Session with Authorization: Bearer header
        - Stores base_url (stripped of trailing slash)

      create_dataset(name: str, metadata: dict) -> tuple[str, dict[str, str]]
        POST /api/datasets
        Body: {"name": name, "metadata": metadata}
        Returns: (dataset_id, {relative_path: signed_upload_url})
        Wraps HTTPError and ConnectionError as GamiphyUploadError.

      upload_file(signed_url: str, file_path: Path) -> None
        PUT signed_url with streaming file body (data=open(file_path, "rb"))
        Does NOT send Authorization header (signed URL is self-authenticated).
        Wraps errors as GamiphyUploadError including the file path in message.

      confirm_dataset(dataset_id: str) -> None
        POST /api/datasets/{dataset_id}/confirm
        Marks dataset as ready/visible in gamiphy dashboard.
        Wraps errors as GamiphyUploadError.

   e) upload_dataset_to_gamiphy(root, repo_id, meta_info, base_url=None) -> str
      Top-level orchestrator. Called from LeRobotDataset.push_to_gamiphy().
      Steps:
        1. get_gamiphy_api_key()
        2. collect_dataset_files(root)
        3. client.create_dataset(name, metadata) — name derived from repo_id
        4. For each file: look up signed URL by relative path, client.upload_file()
           Log: INFO "Uploading [N/M] <relative_path>"
        5. client.confirm_dataset(dataset_id)
      Returns dataset_id. Uses logging.getLogger(__name__) throughout.

2. tests/utils/test_gamiphy.py
   Unit tests using unittest.mock. Covers:
   - Auth: missing key, empty key, present key
   - File collection: excludes images, stable order, multiple patterns
   - create_dataset: success, HTTP error (401), network error (ConnectionError)
   - upload_file: success (streaming body), missing URL in map, PUT error
   - confirm_dataset: success, server error (500)
   - Full pipeline: tmp_path with synthetic files, all three HTTP calls mocked,
     assert correct call order and arguments
   - Fast-fail: no HTTP calls made when API key is missing

     def push_to_gamiphy(self, base_url: str | None = None) -> str:
         """Upload dataset to gamiphy.ai. Reads auth from GAMIPHY_UPLOAD_KEY env var."""

             repo_id=self.repo_id,
             meta_info=self.meta.info,

Key Design Decisions
--------------------

- push_to_gamiphy defaults to False (opt-in), unlike push_to_hub (True by default)
- API key is NOT a config field — always from env var (no secrets in CLI history
  or config files)
- gamiphy_base_url config field allows staging/self-hosted override without code
  changes
- Lazy import in push_to_gamiphy() — zero import cost for users who don't use it
- Streaming file upload (data=open(file_path, "rb")) — avoids loading large video
  files into memory
- No progress-bar dependency — plain logging.INFO with [N/M] counters follows
  existing lerobot conventions
- gamiphy failure does NOT roll back local data or HF Hub upload (independent)
- dataset name sent to gamiphy = the part of repo_id after "/" (or full repo_id
  if no slash)


Upload Flow Summary
-------------------
User sets GAMIPHY_UPLOAD_KEY env var
  │
  ▼ lerobot record --dataset.push_to_gamiphy=true ...
  │
  ▼ record() completes all episodes
  │
  ▼ dataset.finalize()          [closes parquet/video writers]
  │
  ▼ dataset.push_to_gamiphy()   [if cfg.dataset.push_to_gamiphy]
    │
    ├─ get_gamiphy_api_key()    [fast fail if GAMIPHY_UPLOAD_KEY not set]
    ├─ collect_dataset_files()  [glob data/ meta/ videos/ - exclude images/]
    ├─ POST /api/datasets       [→ dataset_id + {path: signed_url} map]
    ├─ PUT <signed_url> × N     [one per file, streaming, ~[1/47]...[47/47] logs]
    └─ POST /api/datasets/{id}/confirm
  │
  ▼ Dataset visible in gamiphy.ai dashboard


Verification Steps
------------------
1. Unit tests: python -m pytest tests/utils/test_gamiphy.py -v
2. Dry-run: Set GAMIPHY_UPLOAD_KEY=test_key, point gamiphy_base_url to a local
   mock server (e.g. httpbin or a simple FastAPI stub), record a minimal dataset
3. End-to-end: Run with --dataset.push_to_gamiphy=true against staging/prod,
   verify dataset appears in gamiphy.ai dashboard with correct metadata
4. Independence test: Run with both push_to_hub=true and push_to_gamiphy=true
   using an invalid gamiphy key; verify HF Hub upload succeeds despite gamiphy
   failure
